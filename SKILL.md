# 香港赛马分析器 (HK Racing Analyzer)

数据驱动的赛马预测工具，结合历史数据分析和AI辅助判断。

## 核心功能

1. **赛事数据分析** - 抓取HKJC赛事数据，分析各参赛马匹
2. **预测分析** - 基于多维度数据给出前3名概率分布
3. **投注建议** - 提供简明的投注建议

## 分析维度

### 主要分析维度（权重较高）

| 维度 | 数据来源 | 分析要点 |
|------|----------|----------|
| **马匹历史战绩** | HKJC马匹档案 | 近5场/10场成绩、同场地表现、同距离表现 |
| **赔率走势** | HKJC赔率页面 | 开盘赔率 vs 临场赔率、赔率变化趋势 |
| **配速/分段** | HKJC赛果页面 | 前段速度、后段冲刺、分段时间 |

### 辅助分析维度（权重较低，次要因素）

| 维度 | 数据来源 | 分析要点 | 权重 |
|------|----------|----------|------|
| **HKJC 贴士指数** | HKJC 官方贴士指数页面 | 官方每日贴士（>100 偏热，<100 偏淡） | **6%** |
| 骑师胜率 | 马匹历史战绩（动态统计） | 该骑师骑乘本马的历史胜率/前3率 | 5% |
| 练马师胜率 | 马匹历史战绩（动态统计） | 该练马师带本马的历史胜率/前3率 | 4% |
| 档位分析 | 历史统计 | 同距离不同档位胜率（沙田/跑马地） | 5% |
| 场地偏好 | 马匹档案 | 草地/泥地、好地/快地表现（含在同条件历史中）| - |
| 负磅/体重 | 赛事数据 | 负磅变化、体重变化趋势（待实现）| - |

### 外部参考维度（辅助参考）

| 维度 | 数据来源 | 分析要点 |
|------|----------|----------|
| **专家预测** | 网络搜索 | 专业马评人、赛马专家的观点和推荐 |

#### 专家预测获取方式

使用 `web_search` 工具搜索专家预测：

```
搜索关键词:
- "沙田赛马 第X场 预测"
- "跑马地 今晚 马评"
- "HKJC 赛马分析"
- "马经 推荐"
```

参考的专家来源：
- 香港赛马会官方马评
- 东方马经、太阳马经等专业媒体
- 知名马评人博客/专栏

专家预测整合方式：
1. 搜索多个专家的预测观点
2. 提取各专家推荐的热门马匹
3. 统计专家共识度（多少专家推荐同一匹马）
4. 作为辅助参考，权重约 5-10%

注意事项：
- 专家预测仅供参考，不盲从
- 重点关注有数据支撑的专家分析
- 警惕过度乐观/悲观的极端观点

## 工作流程

### Step 1: 解析用户请求

从用户输入中提取：
- 赛事日期（默认今天）
- 场地（沙田/跑马地）
- 场次号

示例：
- "分析今天沙田第3场" → 日期=今天, 场地=沙田, 场次=3
- "分析跑马地今晚第5场" → 日期=今天, 场地=跑马地, 场次=5

### Step 2: 获取赛事数据（带缓存）

脚本内置磁盘缓存，**避免对同一场次重复抓取**，节省网络请求和等待时间。

缓存存储位置：
```
<skill目录>/.cache/<url_hash>.json
```

各类数据的缓存 TTL：

| 数据类型 | TTL | 说明 |
|----------|-----|------|
| 历史赛果（已结束） | 7 天 | 赛果不变，长期缓存 |
| 当日赛前排位表 | 30 分钟 | 可能临时换马/换骑 |
| 马匹历史战绩 | 24 小时 | 每场赛后更新一次 |
| 赔率数据 | 5 分钟 | 临场实时变化，短期缓存 |
| 其他通用页面 | 1 小时 | 兜底 TTL |

使用 `web_fetch` 工具访问 HKJC 页面（脚本会自动处理缓存，无需手动干预）：

```
赛事信息页面:
https://racing.hkjc.com/racing/information/Chinese/Racing/LocalResults.aspx

马匹档案页面:
https://racing.hkjc.com/racing/information/Chinese/Horse/Horse.aspx?HorseId=HK_XXXX_XXXX

赔率页面:
https://racing.hkjc.com/racing/information/Chinese/Racing/Odds.aspx
```

### Step 3: 数据分析

#### A. 本地数据分析
运行 `scripts/analyze_race.py` 脚本进行分析：

```bash
python scripts/analyze_race.py --date YYYY/MM/DD --venue ST/HV --race N
```

脚本会：
1. 抓取参赛马匹信息
2. 获取每匹马的历史战绩
3. 获取赔率数据
4. 计算综合评分

#### B. 专家预测获取（可选）
使用 `web_search` 搜索专家预测：

```bash
# 搜索示例
web_search "沙田赛马 第3场 预测 2026年3月"
web_search "跑马地 今晚 马评 推荐"
```

提取信息：
- 专家推荐的热门马匹
- 专家分析的关键理由
- 专家共识度统计

#### C. 综合预测
将本地数据分析与专家预测整合：

```
最终评分 = 本地分析评分 × 90% + 专家共识评分 × 10%
```

专家共识评分计算：
- 统计推荐该马的专家数量
- 除以总专家数量 = 共识度百分比
- 转换为 0-100 分

### Step 4: 生成预测报告

输出格式：

```markdown
## 🏇 [日期] [场地] 第X场 分析报告

### 赛事基本信息
- 赛事名称
- 距离/班次/场地状况
- 奖金

### 📊 前3名概率预测

| 排名 | 马号 | 马名 | 胜出概率 | 分析理由 |
|------|------|------|----------|----------|
| 1 | X | XXX | XX% | ... |
| 2 | X | XXX | XX% | ... |
| 3 | X | XXX | XX% | ... |

### 🎯 专家预测参考

| 马匹 | 专家共识度 | 代表性观点 |
|------|------------|------------|
| XXX | X/X (XX%) | ... |
| XXX | X/X (XX%) | ... |

### 💡 投注建议

**推荐独赢**: X号 XXX
- 理由: ...

**推荐连赢**: X, X
- 理由: ...

**冷门关注**: X号
- 理由: ...

### ⚠️ 风险提示
- ...
```

## 快速开始

当用户请求分析时：

```bash
# 分析今天沙田第3场（自动使用缓存，第二次运行极速返回）
python scripts/analyze_race.py --venue ST --race 3

# 分析指定日期跑马地第5场
python scripts/analyze_race.py --date 2026/03/30 --venue HV --race 5

# 强制刷新：忽略缓存重新抓取所有数据（临场赔率更新时使用）
python scripts/analyze_race.py --venue ST --race 3 --force-refresh

# 清除当前场次缓存后重新分析
python scripts/analyze_race.py --venue ST --race 3 --clear-cache

# 查看缓存统计（文件数 / 总大小 / 最旧条目）
python scripts/analyze_race.py --cache-stats
```

### 缓存使用建议

| 场景 | 推荐操作 |
|------|---------|
| 赛前首次分析 | 直接运行，自动建立缓存 |
| 1小时内重复分析同场次 | 直接运行（缓存命中，秒级完成）|
| 临场前30分钟刷新赔率 | `--force-refresh` 或 `--clear-cache` |
| 分析历史赛事（回测） | 直接运行，赛果永久缓存 |

## 注意事项

1. **数据时效性** - 赔率数据实时变化，分析结果仅供参考
2. **历史不代表未来** - 过往战绩仅供参考，不保证未来表现
3. **风险提示** - 赛马投注有风险，请理性参与
4. **数据来源** - 所有数据来自香港赛马会官网 (HKJC)

## 自动化任务

| 任务名 | 时间 | 功能 |
|--------|------|------|
| 赛马日预测 | 每日 14:30 | 检测明天是否赛马日，若是则批量预测所有场次 |
| 赛马回测+进化分析 | 每日 09:00 | 抓取昨日实际赛果，对比预测，生成进化建议报告 |

## 进化建议工作流

```
[每日 14:30] 检测赛马日 → 批量预测 → 保存存档
                ↓（次日 09:00）
[每日 09:00] 抓取实际赛果 → 对比预测 → 生成进化建议报告
                ↓（用户审阅后）
[手动确认]  python apply_evolution.py --list
            python apply_evolution.py --report .evolution/xxx.md --apply N
                ↓（应用后）
[自动备份]  .backups/analyze_race_YYYYMMDD_HHMMSS.py
            若效果不好：python apply_evolution.py --rollback
```
