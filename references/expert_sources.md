#### HKJC 贴士指数说明
- **数据来源**：HKJC 官方网站每日发布的贴士指数
- **指数范围**：通常 80-120，100 为基准值
  - > 100：市场偏热（被看好）
  - < 100：市场偏冷（被看淡）
- **权重设置**：**5%**（官方每日信号，与赔率共识互补）
- **评分规则**：
  - 指数 100 → 评分 50（中性）
  - 每偏离 5 个点 → 评分 ±10 分
  - 无数据时 → 默认 50 分

> ⚠️ **v1.4.0 更新**：贴士指数从原来的 fetch.py 独立抓取升级为 config.py 常量配置 URL，fetch.py 中统一调用。新马/降班马场景下权重会有适应性调整（详见 `references/analysis_weights.md` 场景权重部分）。

## 专家预测评估标准

### 可靠专家特征
- 有长期赛马分析经验
- 分析有理有据，有数据支撑
- 观点客观，不夸大
- 历史预测准确率较高

### 需警惕的特征
- 过度乐观/只推荐热门
- 没有分析理由的简单推荐
- 频繁改变观点
- 缺乏专业背景

## 专家共识度计算

```python
def calculate_expert_consensus(horse_id, expert_recommendations):
    """
    horse_id: 马匹编号
    expert_recommendations: 专家推荐列表
    返回: 共识度百分比 (0-100)
    """
    total_experts = len(expert_recommendations)
    recommended_count = sum(1 for rec in expert_recommendations if horse_id in rec)
    
    return (recommended_count / total_experts) * 100
```

## 整合专家预测

1. **收集**: 从多个来源收集专家预测
2. **去重**: 合并相同专家的重复推荐
3. **统计**: 计算每匹马的专家共识度
4. **转换**: 将共识度转换为评分 (0-100)
5. **整合**: 按权重 5% 合并到最终评分

## 注意事项

- 专家预测仅供参考，不盲从
- 优先参考有数据支撑的分析
- 结合自身分析做出判断
- 赔率是更可靠的客观指标
