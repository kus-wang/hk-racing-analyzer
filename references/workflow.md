## 报告结构模板

按照以下结构输出 Markdown 报告（参考 `output.py` 的格式）：

```markdown
## [日期] [场地] 第X场 分析报告

### 赛事基本信息
- 赛事名称 / 距离 / 班次 / 场地状况 / 奖金

### 📊 前3名概率预测
| 排名 | 马号 | 马名 | 胜出概率 | 分析要点 |
|:--:|:--:|:--|:--:|:--|
| 1 | X | XXX | XX% | ... |
| 2 | X | XXX | XX% | ... |
| 3 | X | XXX | XX% | ... |

### 🎯 投注风格定位

由 `classify_betting_style()` 函数自动判断（见 `output.py`）：

| 风格 | 含义 | 推荐策略 |
|------|------|----------|
| **🛡️ 稳妥型** | Top1 或前二优势明显 | 优先独赢/位置，控成本 |
| **⚡ 进取型** | 格局开放，无明显热门 | 关注冷门搏高赔，或 PQ/三重彩 |
| **🔀 纠结型** | 多匹马实力接近 | PQ 三组全包，分散风险 |

### 📋 全场分项评分
（各维度 0-100 分，供参考）

### 🔍 冷门关注
（有同条件前3记录、赔率未拉长、班次适配度高的非热门马）

### 💡 投注建议
（见下方投注推荐逻辑）

### ⚠️ 风险提示
```

## 投注推荐逻辑

> **v1.5.0 更新**：投注推荐已由 `betting.py` 模块自动化，取代手动规则。下方说明模块判断逻辑供参考。

### 投注推荐核心判断（`betting.py`）

| 判断维度 | 逻辑 |
|---------|------|
| **场型识别** | 稳妥场（Top1>35%）、进取场（Top1<20%）、纠结场（多马接近）|
| **价值指数** | 期望值 = 模型概率 × 赔率，>1.0 为价值注 |
| **冷门接口** | 赔率>15 + 同条件前3 + 赔率未拉长 → 关注 |
| **回测命中** | betting_recommendation 命中验证，≥3场样本触发命中率统计 |

### 投注方式适用条件（`betting.py` → `get_betting_recommendation()`）

| 玩法 | 条件 |
|------|------|
| **独赢（WIN）** | Top1 概率明显领先（>35%），赔率 ≤4 倍 |
| **位置（PLACE）** | Top3 中任意一匹概率 ≥20% |
| **连赢（Q）** | Top1+Top2 概率和 ≥55%，两者实力接近 |
| **位置连赢（PQ）** | Top3 三匹马概率分散，难以排序 |
| **三重彩（TRIO）** | Top3 均有一定概率（每匹≥10%），三匹和≥45% |

### 投注建议输出格式

报告中的 `### 💡 投注建议` 由 `betting.py` 自动生成，包含：

- 🏆 **本场最推荐**：[玩法] — [马号组合] + 期望值
- 冷门关注马匹（如有）
- 回测命中率（如有 ≥3 场样本）
- 各投注方式参考（WIN/PLACE/Q/PQ/TRIO）

## CLI 用法

入口：`python scripts/analyze_race.py`

```bash
# 分析今天沙田第3场（自动使用缓存，第二次运行极速返回）
python scripts/analyze_race.py --venue ST --race 3

# 分析指定日期跑马地第5场
python scripts/analyze_race.py --date 2026/03/30 --venue HV --race 5

# 强制刷新：忽略缓存重新抓取所有数据（临场赔率更新时使用）
python scripts/analyze_race.py --venue ST --race 3 --force-refresh

# 清除当前场次缓存后重新分析
python scripts/analyze_race.py --venue ST --race 3 --clear-cache

# 查看缓存统计（文件数 / 总大小 / 最旧条目）
python scripts/analyze_race.py --cache-stats
```

### 缓存使用建议

| 场景 | 推荐操作 |
|------|---------|
| 赛前首次分析 | 直接运行，自动建立缓存 |
| 1小时内重复分析同场次 | 直接运行（缓存命中，秒级完成）|
| 临场前30分钟刷新赔率 | `--force-refresh` 或 `--clear-cache` |
| 分析历史赛事（回测） | 直接运行，赛果永久缓存 |

## 自动化调度

入口：`python scripts/daily_scheduler.py`

```bash
# 每日 14:30 预测任务：检测明天是否为赛马日 → 有则批量预测所有场次
python scripts/daily_scheduler.py --mode predict

# 每日 23:30 回测任务：加载今天/昨天的预测存档 → 抓取实际赛果 → 生成进化报告
python scripts/daily_scheduler.py --mode backtest
```

### 自动化任务说明

| 任务 | 时间 | 触发条件 | 输出 |
|------|------|---------|------|
| `--mode predict` | 14:30 | 明天为赛马日 | 批量预测存档 → `.archive/YYYY-MM-DD_{venue}.json` |
| `--mode backtest` | 23:30 | 当日有预测存档 | 进化报告 → `.evolution/YYYY-MM-DD_{venue}_report.md`，完成后自动归档至 `.archive/completed/` |

> **归档隔离机制**：回测完成后，预测存档自动从 `.archive/` 移至 `.archive/completed/`，防止重复加载或污染后续回测任务。

## 马匹对比功能

当用户要求对比两匹（或多匹）马时，执行以下分析流程：

### Step 1：识别对比请求

用户表述如"比较3号和7号"、"X号好还是Y号好"、"3号7号哪个更有机会"均识别为对比请求。

### Step 2：抓取评分数据

调用 `analyze_race.py` 获取全场评分，从结果中提取对比马匹的各维度分项数据。

### Step 3：逐维度对比分析

| 对比维度 | 说明 |
|----------|------|
| 综合评分 | 总分高低 |
| 历史战绩 | 同条件/同场地胜率对比 |
| 赔率走势 | 资金流向对比（赔率缩 vs 拉长） |
| 配速匹配 | 跑法与赛道状况匹配度 |
| 骑师/练马师 | 组合实力对比 |

### Step 4：输出对比报告

```markdown
## [场地] 第X场 — 马匹对比：X号 vs X号

### 一句话结论
[根据评分差距给出快速判断]

### 维度对比表
| 维度 | X号 [马名] | X号 [马名] | 优势方 |
|:-----|:--------:|:---------:|:------:|
| 综合评分 | XX | XX | X号 |
| 同条件历史 | XX | XX | X号 |
| 赔率走势 | XX | XX | X号 |
| ... | ... | ... | ... |

### 各维度详细分析
[简要说明每个维度的差距原因]

### 综合建议
[结合概率数据，给出投注方向的建议]
```

## 赛果查询功能

当用户查询当天已结束赛事的赛果时：

### 数据获取

```python
from fetch import fetch_race_results
from parse import parse_race_results

html = fetch_race_results("2026/04/01", "ST")
races = parse_race_results(html)
# races = [{"race_no": 1, "results": [{"pos": 1, "no": "12", "name": "爆熱", ...}]}, ...]
```

### 识别逻辑
- 赛程已结束（当日 23:00 后自动可用）
- 提取各场次实际名次、马号、马名、完成时间、独赢赔率

### 赛果输出格式

```markdown
## [日期] [场地] 赛果

| 场次 | 冠军 | 亚军 | 季军 | 时间 |
|:----:|:----:|:----:|:----:|:----:|
| 1 | X号 [马名] | X号 [马名] | X号 [马名] | HH:MM |
| 2 | ... | ... | ... | ... |
```

如当日赛程尚未结束，告知用户赛果预计公布时间。
