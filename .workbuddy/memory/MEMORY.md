## 项目基础

- **技能名称**：hk-racing-analyzer（香港赛马分析预测）
- **核心脚本**：`scripts/daily_scheduler.py`（调度） + `scripts/analyze_race.py`（预测）
- **自动化任务**：
  - `14-30`：每日 14:30 运行，`--mode predict` 检测明天是否赛马日
  - `23-30`：每日 23:30 运行，`--mode backtest` 回测当天赛果（原 09-00 已改为 23-30）

## 自动化执行规律

- `--mode predict`：检测**明天**是否赛马日，若是则批量预测所有场次并保存 `.archive/`
- `--mode backtest`：加载今天/昨天的预测存档，抓取实际赛果，生成进化报告到 `.evolution/`，**完成后自动归档到 `.archive/completed/`**
- 检测逻辑：依次检测沙田（ST）和跑马地（HV），通过 HKJC 排位表页面确认

## 归档隔离机制（2026-04-01 更新）

- 回测完成后，预测存档自动从 `.archive/` 移至 `.archive/completed/`
- 防止已完成的预测存档被后续回测任务重复加载或污染
- `load_prediction_archive()` 函数优先从原位置加载，若已归档则从 completed 目录加载

## 注意事项

- HKJC 页面抓取需要约 10-20 秒（每个场地各一次 HTTP 请求）
- 脚本使用缓存机制（`.cache/`），赛事排位表 TTL=30 分钟，**赛果 TTL=7天**
- 非赛马日时脚本正常退出（exit 0），automation memory 正常记录
- **回测任务时间**：23:30 确保覆盖沙田日间赛、黄昏赛和跑马地夜赛（通常 22:30-23:00 结束）

## 已知 Bug 及修复记录

### `_parse_result_html` 解析失败（2026-04-02 修复）
- **症状**：全部场次报「⚠ 解析失败」
- **根因 1**：`.cache/` 中有旧的空结果 HTML（赛事未结束时抓取，TTL=7天），重复读取
- **根因 2**：旧正则假设 `<td>名次</td><td>马号</td>` 在同一行，实际为多行嵌套结构
- **修复**：改用逐 `<tr>` 块解析，提取所有 `<td>` 文本，按列位置匹配名次/马号/马名
- **排查方法**：若 backtest 报解析失败，先检查缓存中 `LocalResults` 的 HTML 长度（空结果≈148KB，正常≈194KB）
