# 香港赛马分析器 (HK Racing Analyzer)

数据驱动的赛马预测工具，结合历史数据分析和AI辅助判断。

## 核心功能

1. **赛事数据分析** - 抓取HKJC赛事数据，分析各参赛马匹
2. **预测分析** - 基于多维度数据给出前3名概率分布
3. **投注建议** - 提供简明的投注建议

## 分析维度

### 主要分析维度

| 维度 | 数据来源 | 分析要点 |
|------|----------|----------|
| **马匹历史战绩** | HKJC马匹档案 | 近5场/10场成绩、同场地表现、同距离表现 |
| **赔率走势** | HKJC赔率页面 | 开盘赔率 vs 临场赔率、赔率变化趋势 |
| **配速/分段** | HKJC赛果页面 | 前段速度、后段冲刺、分段时间 |

### 次要分析维度

| 维度 | 数据来源 | 分析要点 |
|------|----------|----------|
| 骑师胜率 | 马匹历史战绩（动态统计） | 该骑师骑乘本马的历史胜率/前3率 |
| 练马师胜率 | 马匹历史战绩（动态统计） | 该练马师带本马的历史胜率/前3率 |
| 档位分析 | 历史统计 | 同距离不同档位胜率（沙田/跑马地） |
| 场地偏好 | 马匹档案 | 草地/泥地、好地/快地表现（含在同条件历史中）|
| 负磅/体重 | 赛事数据 | 负磅变化、体重变化趋势（待实现）|

> 上述维度权重由 `weights.py` 根据场地/距离/班次动态计算，无需记固定数值。

### 外部参考（可选）

使用 `web_search` 搜索专家马评，提取热门马匹和共识观点，作为辅助参考。

搜索关键词：`"沙田赛马 第X场 预测"` / `"跑马地 今晚 马评"` / `"马经 推荐"`

> 专家预测仅供参考，不盲从；重点关注有数据支撑的分析。

## 工作流程

### Step 1: 解析用户请求

从用户输入中提取：
- 赛事日期（默认今天）
- 场地（沙田/跑马地）
- 场次号
- 功能意图（预测 / 对比 / 赛果查询）

**日期自然语言支持**：

| 用户表述 | 解析结果 |
|----------|----------|
| "今天" | 当前日期 |
| "明天" | 当前日期 + 1天 |
| "明天下午" / "明晚" | 当前日期 + 1天 |
| "下周X" / "礼拜X" | 本周指定星期几 |
| "2026/04/05" | 指定日期 |
| "4月5日" | 当前年份指定月日 |

**场地中文映射**：

| 用户表述 | 解析结果 |
|----------|----------|
| 沙田 / ST / 沙田马场 | ST |
| 跑马地 / HV / 跑马地马场 / 今晚 | HV |

**功能意图识别**：

| 用户表述关键词 | 识别为 |
|--------------|--------|
| "分析"、"预测"、"推荐" | 预测分析 |
| "比较"、"对比"、"哪个好"、"X号和Y号" | 马匹对比 |
| "赛果"、"结果"、"今天跑完了吗" | 赛果查询 |

**完整解析示例**：
- "分析今天沙田第3场" → 日期=今天, 场地=ST, 场次=3, 意图=预测
- "分析跑马地今晚第5场" → 日期=今天, 场地=HV, 场次=5, 意图=预测
- "比较3号和7号" → 日期=今天, 意图=对比, 马号=[3, 7]
- "今天沙田第2场谁赢了" → 日期=今天, 场地=ST, 场次=2, 意图=赛果

### Step 2: 获取赛事数据（带缓存）

脚本内置磁盘缓存，**避免对同一场次重复抓取**，节省网络请求和等待时间。

缓存存储位置：
```
<skill目录>/.cache/<url_hash>.json
```

各类数据的缓存 TTL：

| 数据类型 | TTL | 说明 |
|----------|-----|------|
| 历史赛果（已结束） | 7 天 | 赛果不变，长期缓存 |
| 当日赛前排位表 | 30 分钟 | 可能临时换马/换骑 |
| 马匹历史战绩 | 24 小时 | 每场赛后更新一次 |
| 赔率数据 | 5 分钟 | 临场实时变化，短期缓存 |
| 其他通用页面 | 1 小时 | 兜底 TTL |

使用 `web_fetch` 工具访问 HKJC 页面（脚本会自动处理缓存，无需手动干预）：

```
赛事信息页面:
https://racing.hkjc.com/racing/information/Chinese/Racing/LocalResults.aspx

马匹档案页面:
https://racing.hkjc.com/racing/information/Chinese/Horse/Horse.aspx?HorseId=HK_XXXX_XXXX

赔率页面:
https://racing.hkjc.com/racing/information/Chinese/Racing/Odds.aspx
```

### Step 3: 数据分析

运行 `scripts/analyze_race.py` 脚本进行分析：

```bash
python scripts/analyze_race.py --date YYYY/MM/DD --venue ST/HV --race N
```

脚本自动完成：
1. 抓取参赛马匹信息（含排位表）
2. 获取每匹马的历史战绩
3. 获取赔率数据
4. 计算综合评分并输出 Softmax 概率分布

如需外部参考，使用 `web_search` 搜索专家马评（见上方"外部参考"）。

### Step 4: 生成预测报告

#### A. 报告结构模板

按照以下结构输出 Markdown 报告（可直接参考 `output.py` 的格式）：

```markdown
## [日期] [场地] 第X场 分析报告

### 赛事基本信息
- 赛事名称 / 距离 / 班次 / 场地状况 / 奖金

### 📊 前3名概率预测
| 排名 | 马号 | 马名 | 胜出概率 | 分析要点 |
|:--:|:--:|:--|:--:|:--|
| 1 | X | XXX | XX% | ... |
| 2 | X | XXX | XX% | ... |
| 3 | X | XXX | XX% | ... |

### 🎯 投注风格定位

由 `classify_betting_style()` 函数自动判断（见 `output.py`），报告自动插入此表：

| 风格 | 含义 | 推荐策略 |
|------|------|----------|
| **🛡️ 稳妥型** | Top1 或前二优势明显 | 优先独赢/位置，控成本 |
| **⚡ 进取型** | 格局开放，无明显热门 | 关注冷门搏高赔，或 PQ/三重彩 |
| **🔀 纠结型** | 多匹马实力接近 | PQ 三组全包，分散风险 |

### 📋 全场分项评分
（各维度 0-100 分，供参考）

### 🔍 冷门关注
（有同条件前3记录、赔率未拉长、班次适配度高的非热门马）

### 💡 投注建议
（见下方投注推荐逻辑）

### ⚠️ 风险提示
```

#### B. 投注推荐逻辑（AI 自主判断）

根据分析报告中的**概率分布**，按以下原则推荐投注方式：

**1. 独赢（Win）**
- 条件：Top1 概率明显领先（35%+），**或** 赔率 <= 4 倍
- 推荐理由：热门马胜出概率显著高于其他选手
- 风险提示：热门马赔率低，回报有限

**2. 位置（Place）**
- 条件：Top3 中任意一匹概率 >= 20%
- 推荐理由：比独赢门槛低，前三名任一进入即可中奖
- 适用场景：热门马优势明显但不确定能否夺冠

**3. 连赢（Quinella）**
- 条件：Top1 + Top2 概率之和 >= 55%，两者实力接近
- 推荐理由：前两名置信度均高且互有竞争力
- 推荐组合：第1名马号 + 第2名马号

**4. 位置连赢（Place Quinella / PQ）**
- 条件：Top3 三匹马概率分散，难以排序
- 推荐理由：前三名难以排序时，PQ 比三重彩成本低
- 推荐组合：第1+2名、第1+3名、第2+3名三组全包

**5. 三重彩（Trio）**
- 条件：Top3 均有一定概率（每匹 >= 10%），三匹概率之和 >= 45%
- 推荐理由：前三名置信度高，覆盖面广
- 推荐组合：第1+2+3名三匹马

#### C. 最推荐投注方式判断

| 概率分布特征 | 最推荐方式 | 说明 |
|-------------|-----------|------|
| Top1 概率 >= 35%，明显领先 | **独赢** 或 **位置** | 热门明确，低风险 |
| Top1+Top2 概率和 >= 55%，前两名稳定 | **连赢** | 稳中求胜 |
| Top3 概率分散（三匹接近）| **PQ** 或 **三重彩** | 分散下注，覆盖面广 |
| Top3 概率和 >= 55%，整体置信度高 | **三重彩** | 高赔率高回报 |
| 概率分布均匀，无明显热门 | **PQ 三组全包** | 降低单押风险 |

#### D. 投注建议输出格式

```markdown
### 💡 投注建议

**🏆 本场最推荐**: [投注方式] — [马号组合]
> 理由：[简要说明推荐逻辑]

**其他参考**:
- **独赢**: X号 [马名]（概率 XX%）— [理由]
- **位置**: X号（概率 XX%）— [理由]
- **连赢**: X号 + X号（概率和 XX%）— [理由]
- **位置连赢（PQ）**: X号+X号 / X号+X号 / X号+X号 — [理由]
- **三重彩**: X号 + X号 + X号 — [理由]
```

> **注意**：以上仅为技术参考，AI 应根据实际概率分布灵活判断，不强制套用规则。脚本（`output.py`）仅输出概率数据，投注判断由 AI 在生成报告时自主完成。

### 附：赛马对比功能

当用户要求对比两匹（或多匹）马时，执行以下分析流程：

**Step 1**：识别对比请求，提取马号列表

用户表述如"比较3号和7号"、"X号好还是Y号好"、"3号7号哪个更有机会"均识别为对比请求。

**Step 2**：抓取两匹马在该场次的评分数据

调用 `analyze_race.py` 获取全场评分，从结果中提取对比马匹的各维度分项数据。

**Step 3**：逐维度对比分析

| 对比维度 | 说明 |
|----------|------|
| 综合评分 | 总分高低 |
| 历史战绩 | 同条件/同场地胜率对比 |
| 赔率走势 | 资金流向对比（赔率缩 vs 拉长） |
| 配速匹配 | 跑法与赛道状况匹配度 |
| 骑师/练马师 | 组合实力对比 |

**Step 4**：输出对比报告

```markdown
## [场地] 第X场 — 马匹对比：X号 vs X号

### 一句话结论
[根据评分差距给出快速判断]

### 维度对比表
:| 维度 | X号 [马名] | X号 [马名] | 优势方 |
|:-----|:--------:|:---------:|:------:|
| 综合评分 | XX | XX | X号 |
| 同条件历史 | XX | XX | X号 |
| 赔率走势 | XX | XX | X号 |
| ... | ... | ... | ... |

### 各维度详细分析
[简要说明每个维度的差距原因]

### 综合建议
[结合概率数据，给出投注方向的建议]
```

### 附：当天赛果查询功能

当用户查询当天已结束赛事的赛果时：

**数据获取**：调用 `fetch_race_results()` 抓取 HKJC 赛果页面，调用 `parse_race_results()` 解析结构化数据。

```python
# 示例（在 analyze_race.py 上下文中）
from fetch import fetch_race_results
from parse import parse_race_results

html = fetch_race_results("2026/04/01", "ST")
races = parse_race_results(html)
# races = [{"race_no": 1, "results": [{"pos": 1, "no": "12", "name": "爆熱", ...]}, ...]}, ...]
```

**识别逻辑**：
- 赛程已结束（当日 23:00 后自动可用）
- 提取各场次实际名次、马号、马名、完成时间、独赢赔率

**赛果输出格式**：

```markdown
## [日期] [场地] 赛果

:| 场次 | 冠军 | 亚军 | 季军 | 时间 |
|:----:|:----:|:----:|:----:|:----:|
| 1 | X号 [马名] | X号 [马名] | X号 [马名] | HH:MM |
| 2 | ... | ... | ... | ... |
```

如当日赛程尚未结束，告知用户赛果预计公布时间。

## 快速开始

当用户请求分析时：

```bash
# 分析今天沙田第3场（自动使用缓存，第二次运行极速返回）
python scripts/analyze_race.py --venue ST --race 3

# 分析指定日期跑马地第5场
python scripts/analyze_race.py --date 2026/03/30 --venue HV --race 5

# 强制刷新：忽略缓存重新抓取所有数据（临场赔率更新时使用）
python scripts/analyze_race.py --venue ST --race 3 --force-refresh

# 清除当前场次缓存后重新分析
python scripts/analyze_race.py --venue ST --race 3 --clear-cache

# 查看缓存统计（文件数 / 总大小 / 最旧条目）
python scripts/analyze_race.py --cache-stats
```

### 缓存使用建议

| 场景 | 推荐操作 |
|------|---------|
| 赛前首次分析 | 直接运行，自动建立缓存 |
| 1小时内重复分析同场次 | 直接运行（缓存命中，秒级完成）|
| 临场前30分钟刷新赔率 | `--force-refresh` 或 `--clear-cache` |
| 分析历史赛事（回测） | 直接运行，赛果永久缓存 |

## 注意事项

1. **数据时效性** - 赔率数据实时变化，分析结果仅供参考
2. **历史不代表未来** - 过往战绩仅供参考，不保证未来表现
3. **风险提示** - 赛马投注有风险，请理性参与
4. **数据来源** - 所有数据来自香港赛马会官网 (HKJC)
