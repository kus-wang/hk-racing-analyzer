# 分析维度权重配置

## 当前权重设置

> 历史战绩拆分为三个子维度，赔率拆分为绝对值与走势两个子维度，合计仍为 100%。

### 基础权重（默认场景）

| 维度 | 子维度 | 权重 | 说明 |
|------|--------|------|------|
| **历史战绩** | 同距离+同场地近5场 | **18%** ↑ | 最精准的参考：同距离同场地成绩（↑ 原15%） |
| | 同场地（不限距离）近5场 | **13%** ↑ | 场地适应性参考（↑ 原10%） |
| | 班次适配度（评分与班次匹配） | **8%** | 升班/降班对竞争力的影响 |
| **赔率** | 临场赔率绝对值 | **15%** | 反映市场当前预期 |
| | 赔率走势（开盘→临场变化幅度） | **13%** | 反映资金流向，缩水越大越值得关注 |
| **配速/分段** | 配速指数（后段冲刺能力）| **15%** | 基于历史跑法推导，pace_index 实测数据待接入 |
| **骑师** | — | **5%** ↓ | 基于历史战绩中骑师×本马的胜率/前3率动态计算（↓ 次要因素，原8%） |
| **练马师** | — | **4%** ↓ | 基于历史战绩中练马师×本马的胜率/前3率动态计算（↓ 次要因素，原7%） |
| **档位** | — | **5%** | 同距离档位胜率统计 |
| **专家预测** | — | **4%** | 多位专家共识度，降权处理（原6%→4%） |
| **合计** | | **100%** | |

> ⚠️ **权重说明**：骑师/练马师已从固定值50改为基于本马历史战绩的动态评分（见下方评分标准）。释放的6%权重补充至历史战绩维度，强化数据导向精准性。

## 场景自适应权重调整

### 跑马地 (HV) 场次
- 档位权重提高：5% → 8%（转弯急，内档优势明显）
- 对应减少：同场地历史战绩 10% → 7%

### 沙田 (ST) 长途赛 (1800m+)
- 配速权重提高：15% → 20%（长途配速策略更重要）
- 对应减少：临场赔率绝对值 15% → 10%

### 泥地赛事
- 增加"泥地适应性"子维度，权重 10%
- 从"同距离同场地"历史战绩 15% → 5%，专门看泥地成绩
- 同场地历史战绩 10% → 0%（全换为泥地相关维度）

### 初出马（新马）
- 同距离同场地历史战绩权重降为 0%（无参考）
- 增加"晨操状态/评分"维度替代，权重 15%
- 班次适配度提高：8% → 15%

### 降班马（班次降落）
- 班次适配度提高：8% → 15%
- 赔率走势提高：13% → 18%（市场对降班马的反应更灵敏）
- 对应减少：专家预测 4% → 0%

### 升班马（班次提升）
- 历史战绩降权：同距离同场地 15% → 8%（高班竞争不同）
- 赔率走势提高：13% → 18%
- 对应减少：专家预测 4% → 0%

### 配速评分（基于历史分段时间）(0-100)

**配速指数计算方式**：
```
配速指数 = 历史最后300米平均时间 / 该途程标准最后300米时间
配速指数 < 1.0 → 后段冲刺能力强（分数高）
配速指数 > 1.0 → 后段冲刺能力弱（分数低）
```

## 概率计算公式（优化版）

### 核心改进：Softmax 归一化 + 概率上限约束

```python
import numpy as np

def calculate_probability(scores, temperature=1.5):
    """
    使用 Softmax 归一化计算概率，避免线性归一化的极端偏差。
    
    参数：
        scores: 各马匹综合评分列表
        temperature: 温度参数，值越大概率越均摊（建议 1.2-2.0）
    
    返回：
        各马匹胜出概率列表（合计 100%）
    """
    scores = np.array(scores, dtype=float)
    
    # Softmax 归一化
    exp_scores = np.exp(scores / temperature)
    probabilities = exp_scores / exp_scores.sum()
    
    # 上限约束：单匹马概率不超过 50%（赛马极少有超过50%胜率）
    probabilities = np.clip(probabilities, 0, 0.50)
    
    # 重新归一化
    probabilities = probabilities / probabilities.sum()
    
    return (probabilities * 100).round(1).tolist()
```

### 综合评分计算（含场景权重）

```python
def get_weights(venue, distance, track_type, race_scenario):
    """
    根据场地、距离、赛道类型和赛事场景返回适合的权重字典。
    
    场景（race_scenario）可选值：
        "normal"      - 普通场次（默认）
        "newcomer"    - 初出马
        "class_down"  - 降班马
        "class_up"    - 升班马
    """
    # 默认权重（最新版本，2026-03-30 更新）
    weights = {
        "history_same_condition": 0.18,   # 同距离+同场地历史战绩（↑ 原0.15）
        "history_same_venue":     0.13,   # 同场地（不限距离）历史战绩（↑ 原0.10）
        "class_fit":              0.08,   # 班次适配度
        "odds_value":             0.15,   # 临场赔率绝对值
        "odds_drift":             0.13,   # 赔率走势变化
        "sectional":              0.15,   # 配速/分段指数
        "jockey":                 0.05,   # 骑师（↓ 次要因素，原0.08，动态评分）
        "trainer":                0.04,   # 练马师（↓ 次要因素，原0.07，动态评分）
        "barrier":                0.05,   # 档位
        "expert":                 0.04,   # 专家预测
    }
    
    # 场景调整
    if race_scenario == "newcomer":
        weights["history_same_condition"] = 0.00
        weights["history_same_venue"]     = 0.00
        weights["class_fit"]              = 0.15
        weights["expert"]                 = 0.00
        # 补回的权重分配给骑师和练马师（初出马这两项更重要）
        weights["jockey"]   = 0.15
        weights["trainer"]  = 0.13
    
    elif race_scenario == "class_down":
        weights["class_fit"]  = 0.15
        weights["odds_drift"] = 0.18
        weights["expert"]     = 0.00
        weights["history_same_condition"] = 0.10
        weights["history_same_venue"]     = 0.07
    
    elif race_scenario == "class_up":
        weights["history_same_condition"] = 0.08
        weights["odds_drift"] = 0.18
        weights["expert"]     = 0.00
        weights["history_same_condition"] = 0.08
        weights["history_same_venue"]     = 0.09
    
    # 场地调整
    if venue == "HV":
        weights["barrier"]              += 0.03
        weights["history_same_venue"]   -= 0.03
    
    # 距离调整
    if distance >= 1800:
        weights["sectional"]  += 0.05
        weights["odds_value"] -= 0.05
    
    # 泥地调整
    if track_type == "dirt":
        weights["history_same_condition"] = 0.05  # 专看泥地成绩
        weights["history_same_venue"]     = 0.00
        weights["class_fit"]             += 0.05
    
    return weights
```

## 数据充足度与信心度

每匹马在生成预测时，附带"数据充足度"标注，提示用户该马评分的可信程度。

| 数据充足度 | 条件 | 说明 |
|------------|------|------|
| ⭐⭐⭐ 高 | 同条件出赛 ≥ 3场，赔率走势数据完整 | 评分可信度高 |
| ⭐⭐ 中 | 同条件出赛 1-2场，或赔率走势缺失 | 评分有一定参考价值 |
| ⭐ 低 | 无同条件历史数据，或初出马 | 评分主要依赖赔率和骑练 |

## 冷门关注筛选逻辑

满足以下**全部条件**的马匹，标记为"冷门关注"：

1. 临场赔率 > 15
2. 历史有同条件（同距离+同场地）**前3名**记录
3. 赔率走势**未拉长**（开盘赔率与临场赔率相差 < 20%）
4. 班次适配度评分 ≥ 50

> 冷门关注不代表推荐投注，仅提示该马有爆冷潜力，请结合其他因素判断。
